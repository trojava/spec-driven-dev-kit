# Projektprinzipien

Diese Regeln gelten für alle Features. Passe sie vor Projektbeginn an.

## 1. Verständlichkeit

Code und Dokumentation sollen für das Team verständlich sein. Einfache
Lösungen werden bevorzugt, solange sie die Anforderungen vollständig erfüllen.

## 2. Nachweisbare Qualität

Jede Anforderung besitzt überprüfbare Akzeptanzkriterien. Änderungen werden
durch angemessene automatisierte oder dokumentierte manuelle Tests geprüft.

## 3. Sicherheit und Datenschutz

Sicherheits- und Datenschutzrisiken werden bereits in Spezifikation und Plan
betrachtet. Geheimnisse werden nicht im Repository gespeichert. Es werden nur
notwendige Daten erhoben und verarbeitet.

## 4. Kleine Änderungen

Die Umsetzung erfolgt in kleinen, prüfbaren Aufgaben. Eine Aufgabe wird erst
abgeschlossen, wenn ihre Prüfung erfolgreich war.

## 5. Nachverfolgbarkeit

Planschritte und Aufgaben verweisen auf die zugehörigen Anforderungen. Ändert
sich eine freigegebene Anforderung, werden Plan und Aufgaben geprüft und
gegebenenfalls angepasst.

## 6. Menschliche Verantwortung

Ein Mensch gibt Spezifikation und Plan vor der Umsetzung frei. Entscheidungen
mit wesentlichen Auswirkungen auf Architektur, Sicherheit, Kosten oder
Benutzer werden nicht allein durch einen KI-Agenten getroffen.

## Fertig bedeutet

Ein Feature ist fertig, wenn:

- alle vereinbarten Anforderungen und Akzeptanzkriterien erfüllt sind,
- alle vorgesehenen Tests erfolgreich ausgeführt wurden,
- keine bekannten kritischen Fehler oder Sicherheitsrisiken offen sind,
- Dokumentation und Spezifikationsdateien den tatsächlichen Stand abbilden,
- ein Mensch das Ergebnis geprüft hat.
